## Escopo geral:

Desenvolver um sistema simples de abertura e gerenciamento de Ordens
de Serviço (OS) utilizando conceitos fundamentais da Programação
Orientada a Objetos (POO), incluindo:

- Criação de classes e objetos
- Utilização de atributos e métodos
- Implementação de métodos construtores e sobrecarga
- Uso de métodos estáticos
- Aplicação de sobreposição de métodos

## Roteiro:
1. Uma empresa de manutenção técnica precisa de um sistema para
gerenciar Ordens de Serviço (OS). Cada OS possui um número
identificador, nome do cliente, descrição do problema e status da
solicitação. O sistema deve permitir:

- a) Criar e exibir Ordens de Serviço (OS).
- b) Utilizar um contador estático para gerar IDs automaticamente.
- c) Criar múltiplos construtores para facilitar a criação de OSs.
- d) Implementar sobrecarga no método de atualização da OS.
- e) Criar uma classe especializada OS Prioritaria que sobrescreva o
método de exibição
